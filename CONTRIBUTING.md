# Contributing to platformkit

Thanks for helping shape the public platformkit product surface.

## Before you open a change

- Use `platformkit-community` for questions, launch feedback, and roadmap discussion.
- Use this repository for flagship product bugs, release-surface gaps, and curated
  cross-repo contract changes.
- If the real change belongs in a split repo such as `platformkit-apps` or
  `platformkit-backend-kit`, make the implementation there first and use this
  repo to update the flagship contract or launch-facing documentation.

## Contribution standards

- Keep repository boundaries explicit.
- Keep pull requests small and reviewable.
- Update public docs when the visible product surface changes.
- Explain why the change belongs in this repository.
- Include local validation notes in the pull request.

## Good first contributions

- Improve flagship documentation clarity
- Tighten product-contract wording
- Fix broken launch-facing links or verification instructions
- Clarify roadmap items and release expectations

## Pull request checklist

- Scope belongs in `platformkit`
- Public docs and links are updated
- Risk or rollback is documented
- Validation steps are included
- Community impact is clear

## Community expectations

By participating in this repository, you agree to the standards in
[CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
