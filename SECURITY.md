# Security Policy

## Scope

Use this channel for vulnerabilities that affect the flagship platformkit
surface, release process, verification tooling, or documentation in a way that
could mislead secure deployment or operation.

## Reporting a vulnerability

- Do not open a public issue for a suspected vulnerability.
- Use GitHub private vulnerability reporting for this repository if it is
  available.
- If you cannot use private reporting, email `security@septagon.dev` with:
  - a concise description of the issue
  - the affected component or document
  - reproduction steps or proof of impact
  - any known mitigations

## What to expect

- We will acknowledge credible reports as quickly as practical.
- We may ask for additional detail or a private reproduction.
- We will coordinate remediation and disclosure timing with the reporter when
  appropriate.

## Out of scope

- Public support questions
- Feature requests
- Issues that belong in another repository without direct impact on the flagship
  surface
